# Contributing

First thank you for wanting to contribute to the Safety wiki.

## Step 1 | Fork the repository

If you dont know how how to fork a repository you can follow this guide:
https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/working-with-forks/fork-a-repo

## Step 2 | Figure out what you want to contribute

Do you want to edit already existing articles, or create a new one, or you dont know what you want to do? All of these are great ways to contribute and if you dont know what you want to do, we can help you out.

### You dont know what you want to contribute

If you do not know what you want to contribute, you can take a look at the [issues page](https://github.com/POBots-353/Safety-wiki/issues). Once you found one you like and is not claimed by someone else, leave a comment saying you would like to work on it. 

### Once you know what you want to work on

#### New article

To create a new article make a new file that ends in `.md` in the folder `Content`. Once that file is made you can start writing!

#### Edit an existing article

First find the correct file in the `Content` folder. Once you found it you can press the edit button (pencil icon) on the top right to edit it!

## Step 3 | Pull Request

One you made your new article or edit, you need to make a Pull Request (PR). This is when a member of the POBots will look at your changes before they get merged (actually changed in the wiki). To create a PR go this this link https://github.com/POBots-353/Safety-wiki/compare

The reason we have this step is to look over the changes for grammar, spelling, format, etc. If we see any issues we will ask you to fix it. Once they are fixed, we will merge it!

## Step 4 | Celebrate

Once your PR is merged its time to celebrate that you contributed to the Safety Wiki!
